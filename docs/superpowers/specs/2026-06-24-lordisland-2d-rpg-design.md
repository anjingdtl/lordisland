## 1. 目标与定位

### 1.1 项目类型
**爱好者的完整可玩作品** —— 不是商业项目，不是单纯学习 demo，是面向 Steam 发布的、流程约 8-12 小时的完整章节 RPG。

### 1.2 故事范围
忠实复刻《罗德岛战记 OVA》动画中**帕恩一行人的主线剧情**，覆盖：
- 序章：帕恩救出艾特、蒂特加入
- 第 1 章：前往 Flaim 城、吉姆加入、龙与帕恩父亲登场
- 第 2 章：马莫边境、火焰精灵之谷、黑色之裔登场、最终决战

不覆盖：第二部"英雄骑士传"和第三部"叛乱的炎之部族"（如果有精力可以在发售后再做 DLC/续作）。

### 1.3 目标平台
**PC（Steam 为主）**，1080p/60fps 目标。Windows 是首要，Linux/macOS 作为加分项（Godot 原生跨平台，后期可加）。

### 1.4 时间预算
**全职 6-12 个月**。具体里程碑估算 30-40 周（7-10 个月），落在预算内。

### 1.5 成功标准
- 一个陌生玩家能从头到尾打通一整周目
- Steam 评价达到"特别好评"以上（≥85% 好评）
- 中英双语完整可用
- 战斗系统平衡，不卡等级也不无聊
- 关键 OVA 场景在游戏中能找到对应

## 2. 体验与设计核心

| 维度 | 选择 |
|---|---|
| 美术风格 | **HD-2D**（2D 像素角色 + 3D 透视场景 + 现代光影） |
| 引擎 | **Godot 4.x**（GDScript 为主，复杂系统辅以 C#） |
| 战斗 | **经典回合制 JRPG** |
| 节奏 | **探索 + 剧情平衡** |
| 上场角色 | **最多 5 人**（5-6 人小队全员可上场） |
| 遇敌 | **可见敌**（无随机遭遇） |
| 昼夜 | **实时昼夜** |
| 存档 | **完全自由**（随时可存、无任何限制） |
| 语言 | **中英双语**（UI、对话、事件、菜单全双语） |
| 音频 | **AI 生成 BGM + 免版税 SFX** |

### 2.1 美术风格定调
- 角色：32×48 像素 sprite 起步，2-4 帧动画
- 场景：3D 低模 + 2D 前景 + 后处理（Outline、DOF、暖色调）
- UI：现代极简 + 中世纪羊皮纸感
- 调色板：偏暖、偏暗、突出战斗特效

### 2.2 引擎与语言选择
- **Godot 4.x**：免费开源、HD-2D 表现力强、2D/3D 混合原生
- **GDScript**：迭代快、语法简单、读起来像伪代码
- **C#（选择性使用）**：仅在需要静态类型安全的复杂系统（战斗公式、存档序列化）

### 3.2 核心系统（10 个）

| # | 系统 | M1 切片目标 |
|---|---|---|
| 1 | **HD-2D 渲染层** | 透视相机 + 1 张测试地图 + 1 个角色可动 |
| 2 | **探索系统** | 1 张洞窟地图、玩家控制器、NPC 交互 |
| 3 | **对话系统** | JSON 驱动 + 分支选项 + 条件触发 |
| 4 | **回合制战斗** | 1v1 → 5v3 全流程，含技能、状态 |
| 5 | **库存/装备** | 至少 5 件物品、1 个装备槽 |
| 6 | **存档/读档** | 自由存档、3-5 个槽位、战斗中可存 |
| 7 | **UI 框架** | 主菜单、HUD、对话框、战斗 UI、设置 |
| 8 | **剧情事件系统** | on_enter_map / on_talk / on_battle_end 触发 |
| 9 | **音频管理** | BGM 淡入淡出、SFX 池、动态切换 |
| 10 | **本地化** | 中英双语可切换、所有字符串走 i18n key |

### 3.3 架构原则
- **数据驱动优先**：对话、物品、敌人、技能、地图事件全部 JSON
- **场景与逻辑分离**：.tscn 放节点结构，业务逻辑在 .gd
- **单例仅用于横切关注点**：GameState / AudioManager / SaveManager / LocaleManager
- **内容与引擎解耦**：所有"游戏内容"在 `data/`，方便后期做 mod

### 3.4 版本控制
- Git + Git LFS（精灵图、音频、字体走 LFS）
- `.gitignore` 包含 `.godot/`、`*.translation`、`addons/*/demo/`
- 备份：周一次到 GitHub 私有仓库

## 4. 主角团与战斗系统

### 4.1 主角团（5-6 人，固定角色，可全员上场）

| 角色 | 职业原型 | 定位 | OVA 设定 |
|---|---|---|---|
| **帕恩** | 战士/勇者 | 前排物理输出、领导核心 | 自由骑士之子 |
| **艾特** | 司祭 | 治疗、辅助魔法 | 玛法神殿司祭 |
| **斯雷因** | 魔法师 | 远程魔法输出、地图解谜 | 高等魔法师 |
| **蒂特** | 精灵弓箭手 | 远程物理、中排输出 | 森林精灵 |
| **石生咡** | 矮人战士 | 前排肉盾、破甲 | 矮人战团成员 |
| **吉姆** | 山贼头领（第 2 章加入） | 高血量前排、突袭 | 灼热沙漠山贼 |

**上场规则**：全部 5-6 人可上场，最多 5 人同时出战。无"换人"系统，全员可上场鼓励玩家尝试不同搭配。

### 4.2 战斗循环
```
[遭遇] → [战斗 UI 淡入] → [行动顺序计算（按速度）]
                                    ↓
                ┌────── [玩家回合] ←─────┐
                │   选：攻击/技能/物品/防御/逃跑    │
                │   → 执行 → 动画/特效          │
                │   → 状态更新                  │
                └────────────────┬──────┘
                                 ↓
                ┌────── [敌方回合] ──────┐
                │   AI 选择 → 执行 → 动画   │
                └────────────────┬──────┘
                                 ↓
                          [胜负判定]
                                 ↓
              [胜利] → 经验/掉落/剧情事件
              [失败] → 战斗结束画面 → 读最近存档
```

### 4.3 属性与成长
- **基础属性**：HP / MP / 力量 / 敏捷 / 智力 / 体力 / 魅力
- **等级**：战斗结束获得经验，曲线由职业模板控制
- **技能**：每角色固定技能树，特定等级解锁，MP 消耗
- **装备**：武器 / 盾 / 头盔 / 铠甲 / 饰品（共 5 槽位）

### 4.4 技能类型（6 类）
1. 物理攻击
2. 物理 buff/debuff（攻击↑ / 防御↓ / 中毒 / 睡眠）
3. 治疗 / 复活
4. 攻击魔法（火、冰、雷）
5. 辅助魔法（祝福 / 圣光 / 沉默敌方）
6. 角色专属剧情技（帕恩"光之剑"、艾特"治愈祈祷"、斯雷因"火球术"等）

### 4.5 状态效果
持续 N 回合（默认 3 回合），优先级管理，免疫与抗性字段。
- 减益：中毒、睡眠、沉默、恐惧、麻痹、致盲
- 增益：攻击↑、防御↑、速度↑、命中↑、回避↑

### 4.6 伤害公式
```
物理伤害 = max(1, (力量 - 防御 * 0.5) * 随机(0.9, 1.1) * 暴击系数)
魔法伤害 = max(1, (智力 * 技能系数 - 抗性) * 随机(0.9, 1.1))
```
数据全放 `data/skills.json`、`data/enemies.json`、`data/characters/*.json`，便于调整平衡。

### 4.7 战斗 UI
- 顶部：行动顺序时间轴
- 右侧：当前角色面板 + 命令菜单
- 底部：技能 / 物品快捷选择
- 伤害数字、暴击、特效用粒子系统
- 战斗背景：HD-2D 透视感（远景 3D 山脉 + 近景 2D tile）

### 4.8 战斗失败处理
- 战斗失败 → 战斗结束画面 → 提示最近存档 → 读档
- 不在战斗中"原地复活"，保持戏剧性

## 5. 世界与探索

### 5.1 地图三类

| 类型 | 用途 | 移动方式 | 丰富度 |
|---|---|---|---|
| **大地图（Overworld）** | 章节间大区域，连接城镇与迷宫 | 自由走动 + 可见敌 | 中等 |
| **城镇地图（Town）** | 剧情对话、商店、住宿、NPC 群 | 自由走动 | **高**（最丰富的探索场所） |
| **迷宫地图（Dungeon）** | 战斗、解谜、Boss | 自由走动 + 强制战斗 | 中等（绑定大地图/城镇环境） |

**绑定原则**：迷宫的视觉/地形/主题必须与大地图或所属城镇的环境特征结合。例如：
- "洛奈城下水道"绑定洛奈城，材质/光源与洛奈城一致
- "Flaim 城外废墟"绑定 Flaim 城附近的灼热沙漠，色调偏红黄
- "火焰精灵之谷"绑定灼热沙漠 + Flaim 城间

### 5.2 世界结构（章节 + 地图清单）

**【序章】洛奈地区**
- 洛奈城（town）—— 帕恩/艾特/斯雷因的初始位置
- 起始洞窟（dungeon，绑定洛奈城）—— 救出艾特，遇蒂特
- 洛奈地区野外（overworld）—— 前往 Flaim

**【第 1 章】Flaim 攻防**
- Flaim 城（town）—— 帕恩一行人抵达
- 灼热沙漠（overworld）—— 吉姆加入
- Flaim 城战场（story 场景）—— 帕恩父亲登场、龙登场

**【第 2 章】马莫边境与黑色之裔**
- 马莫边境要塞（town）—— 偶遇卡迪斯
- 火焰精灵之谷（dungeon，绑定灼热沙漠）—— 寻找火焰精灵
- 黑色之裔据点（dungeon）—— 最终章
- 最终决战（story 场景）—— 卡拉 VS 贝鲁特

### 5.3 视角与镜头
- 默认 **3/4 俯视**（约 45° 角），HD-2D 标准
- 战斗同视角，无缝切换（不切战斗背景）
- 镜头跟随主角，距离可调
- 重要演出：固定镜头 + 角色朝向 + 字幕

### 5.4 实时昼夜系统
- 主角活动期间按真实时间推进：**10 分钟现实时间 = 30 分钟游戏时间**（即现实 1 分钟 = 游戏 3 分钟）
- 完整的"白天 → 黄昏 → 夜晚 → 黎明 → 白天"循环 = 8 小时现实时间
- 白天 / 黄昏 / 夜晚 / 黎明四个时段，自动循环
- 不同时段的视觉效果：
  - **白天**：明亮、暖色、远景清晰
  - **黄昏**：橙红、远景压暗、近景暖光
  - **夜晚**：冷色、灯笼/火光、远景压抑
  - **黎明**：蓝紫、露水效果
- NPC 行为绑 `time_of_day`：白天在市集，夜晚在酒馆/家
- 商店营业时间：大部分白天/黄昏营业，夜晚关闭（神庙、旅馆 24h）
- 城镇地图：每个时段有独立的灯光图集（4 套）

### 5.5 遇敌设计
- **大地图 / 迷宫**：可见敌（小图标游荡），接触触发战斗
- **迷宫深层**：分步遇敌（每走 N 步一次遭遇概率）
- 战斗胜利后该敌消失（小区域清除机制）
- **城镇内不遇敌**（安全区）

### 5.6 交互对象
- NPC：靠近按 E 触发对话
- 宝箱：触发开箱动画 + 物品入包
- 出口/入口：触发切场 + 转场动画
- 隐藏物：调查触发，依赖魅力/感知属性

### 5.7 移动与节奏
- 主角 200 px/s，奔跑 1.5x
- 城镇/野外无限制奔跑
- 迷宫中"走路模式"为主（紧张感），遇敌前镜头轻推
- 切场：淡入淡出 0.5s

## 6. 剧情内容管线

### 6.1 内容资产类型

| 类型 | 格式 |
|---|---|
| 对话树 | `data/dialogues/<id>.json` |
| 章节事件 | `data/events/<chapter>_<id>.json` |
| NPC 定义 | `data/npcs/<id>.json` |
| 物品 | `data/items.json`（全局） |
| 敌人 | `data/enemies.json`（全局） |
| 技能 | `data/skills.json`（全局） |
| 地图 | `scenes/world/<map_id>.tscn` |

### 6.2 对话系统（JSON 驱动）

```json
// data/dialogues/npc_ehto_intro.json
{
  "id": "npc_ehto_intro",
  "speaker": "ehto",
  "portrait": "ehto_neutral",
  "nodes": [
    { "id": "start", "text_key": "ehto_intro_1", "next": "ask_who" },
    { "id": "ask_who", "text_key": "ehto_intro_2",
      "choices": [
        { "label_key": "choice_a", "next": "answer_a" },
        { "label_key": "choice_b", "next": "answer_b" }
      ] },
    { "id": "answer_a", "text_key": "ehto_intro_3", "next": "end" },
    { "id": "answer_b", "text_key": "ehto_intro_4", "next": "end" },
    { "id": "end", "type": "end" }
  ]
}
```

- 文本走 i18n key，存 `locale/zh.po` 和 `locale/en.po`
- 支持：分支选项、条件（`requires_flag` / `requires_item`）、动作（`give_item` / `set_flag` / `start_battle`）
- 支持：情绪（中立/高兴/悲伤/愤怒）→ 切换立绘表情
- 字体：中英双字体自动 fallback

### 6.3 事件系统（章节脚本）

```json
// data/events/ch1_flaim_attack.json
{
  "id": "flaim_attack",
  "trigger": { "type": "on_enter_map", "map": "flaim_town" },
  "conditions": [ { "flag": "ch1_started" } ],
  "steps": [
    { "type": "show_dialogue", "id": "ghim_warning" },
    { "type": "play_cutin", "asset": "dragon_appears" },
    { "type": "start_battle", "enemies": ["dragon", "soldier_a", "soldier_b"] },
    { "type": "on_victory", "set_flag": "ghim_rescued" },
    { "type": "show_dialogue", "id": "ghim_thanks" }
  ]
}
```

事件触发器类型：
- `on_enter_map` / `on_leave_map`
- `on_talk`（与 NPC 对话后）
- `on_flag_set`（某个 flag 被设置时）
- `on_item_used`（使用某物品）
- `on_battle_end`（战斗结束）
- `on_time_of_day`（进入特定时段）

### 6.4 章节脚本流程

```
[章节开始] → set flag: <chapter>_started
            ↓
[章节内场景/事件循环]
            ↓
[章节节点判定]（N 个关键事件）
            ↓
[章节完成] → set flag: <chapter>_done
            ↓
[触发下一章 prologue]
```

### 6.5 NPC 设计

**每个城镇 5-15 个 NPC**：
- 主线 NPC（4-5 个）：推进剧情
- 支线 NPC（3-5 个）：支线任务、对话、彩蛋
- 氛围 NPC（2-5 个）：闲聊、商店店主、酒馆

**NPC 行为依赖 time_of_day**：
- 白天：市集、街道
- 黄昏：酒馆、回家路上
- 夜晚：酒馆、家中、隐藏剧情
- 黎明：早起、特殊对话

**关键对话可挂"魅力判定"**：高魅力看到额外信息。

### 6.6 演出系统（Cutscene）
- 简单脚本：相机机位、角色朝向、字幕、淡入淡出
- 不做复杂运镜
- 关键演出用预渲染 + 字幕（黑底立绘+文字）作为兜底
- 战斗关键一击用放大特写

### 6.7 作者工具
- 数据用 JSON，不做 GUI 编辑器
- 用 Godot 内置 Inspector 配 NPC 触发器
- 章节流程图用 Mermaid / draw.io（仅前期规划）

## 7. 存档系统

### 7.1 存档规则
- **完全自由**：玩家随时可存，无任何限制
- 战斗中可存档（暂停时弹出存档菜单）
- 任意场景可存档（包括剧情过场，按 Esc 暂停即可）
- 不允许"防作弊存档"——单机游戏不需要

### 7.2 存档内容
- 玩家位置（地图 + 坐标 + 朝向）
- 队伍状态（HP/MP/Buff）
- 队伍组成（哪些角色在队）
- 库存（物品 + 数量 + 装备）
- 章节进度（所有 flag）
- 已阅读对话记录（避免重复弹同段对话）
- time_of_day
- BGM 状态（恢复时无缝衔接）

### 7.3 存档槽位
- 多个手动存档槽（建议 10 个）+ 1 个自动存档（章节切换时）
- 存档列表显示：时间戳、主角 Lv、所在章节、所在地图、缩略图（场景截图）

## 8. 资源管线

### 8.1 美术
- **角色 sprite**：32×48 像素 / 4 方向 × 4 帧
  - 主角 5-6 人 × 4 方向 × 4 帧 = 80-96 张
  - 敌人 20-30 种 × 4 方向 × 2-3 帧 = 200-300 张
  - 部分由 AI 工具生成 + 人工调色 + 加描边
- **场景 tile**：16×16 基础 + 32×32 装饰
  - 大地图/城镇/迷宫各一套
- **UI**：羊皮纸风格组件库
- **光影后处理**：Outline + 暖色调滤镜

### 8.2 音频
- **BGM**：AI 工具（Suno / Udio）生成，按场景分类
  - 城镇（5-8 首）
  - 野外/大地图（3-5 首）
  - 迷宫/战斗（5-8 首）
  - 剧情/重要演出（3-5 首）
  - 主菜单 / 标题（1-2 首）
- **SFX**：免版税音效包 + 关键场景定制
  - 菜单 / 移动 / 攻击 / 魔法 / 物品 / 获得物品
  - 环境音（洞窟风声、沙漠风声、城镇喧嚣）

### 8.3 字体
- 中文：思源黑体 / 霞鹜文楷
- 英文：Google Fonts 上找一款中世纪风格（如 Cinzel、IM Fell English）

## 9. 国际化

- **中英双语**（必须）
- 不包含日语（节省翻译工作量；如有海外玩家需求可作为后期 DLC）
- 所有可见字符串走 i18n key
- 字体自动 fallback（中文为主，英文为辅）
- 切换语言无需重启游戏
- 后期可加入德语/法语等，但不在 6-12 个月预算内

## 12. 风险与对策

| 风险 | 等级 | 对策 |
|---|---|---|
| 美术资源产出慢，独立开发跟不上 | 高 | AI 工具生成 + 极简 sprite 规格 + 委托关键角色立绘 |
| 战斗平衡反复调整 | 中 | 数据驱动 + 早期就开始 alpha 测试 |
| HD-2D 渲染性能问题 | 中 | 早期做性能原型（1 张测试地图 100 个 sprite 跑满 60fps） |
| 中英双语翻译耗时 | 中 | 关键文本先中文，后期再翻英文；关键章节优先 |
| 实时昼夜系统对城镇美术工作量翻倍 | 高 | M3 才上线，城镇美术在 M2 末开始 |
| 个人开发动力/精力问题 | 高 | 里程碑切小（4-5 周一段），每段结束都有可玩 demo 提振士气 |
| 引擎功能踩坑 | 中 | M1 末尾安排"风险评估周"，识别引擎限制 |

## 14. 关键决策记录

| 决策点 | 选择 | 备选 | 理由 |
|---|---|---|---|
| 项目定位 | 爱好者完整作品 | 学习/商业/原型 | 中等范围、可发布、不赶进度 |
| 故事范围 | 帕恩一行人主线 | 序章/三部曲/原创 | 与 6-12 月预算匹配 |
| 美术风格 | HD-2D | 像素/二次元/插画 | Steam 辨识度、技术成熟、美术可控 |
| 引擎 | Godot | Unity/Unreal/自研 | HD-2D 生态最好、免费、跨平台 |
| 战斗 | 回合制 | ATB/动作/SRPG | 与 OVA 气质匹配、平衡性好调 |
| 上场人数 | 5 | 4 | 原作 5-6 人小队 |
| 遇敌方式 | 可见敌 | 随机 | 现代 JRPG 惯例 |
| 昼夜 | 实时 | 固定 | 沉浸感 |
| 存档 | 完全自由 | 章节点 | 玩家便利 |
| 语言 | 中英 | 仅中/三语 | 30-50% 增量受众 |
| 音频 | AI + 免版税 | 全原创/全免版税 | 性价比高 |
| 路线 | 垂直切片 | 战斗优先/世界优先 | 风险最低 |
